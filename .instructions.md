# Ramy's Brain — Auto-Capture Instructions

You have access to the `ramym-brain` MCP server. Use it **automatically** without being asked — treat it as your persistent memory layer.

## Auto-Capture Rules

### Always store decisions
After ANY architectural decision, technology choice, or design trade-off in this conversation:
- Call `log_decision` with the decision, context, and alternatives considered
- Examples: choosing a library, picking an approach, deciding on a pattern

### Always store errors
When ANY error is encountered and resolved:
- Call `log_error` with the error message, context, and resolution
- This builds a personal knowledge base of fixes

### Always store key learnings
When something non-obvious is discovered or confirmed:
- Call `remember` with the insight
- Examples: "Supabase pgvector needs ivfflat index with at least 100 rows", "Flash model is 3x faster for planning endpoints"

### Always store code patterns
When a reusable pattern or snippet is created or discussed:
- Call `save_snippet` with the code, language, and descriptive tags

### Person interactions
When discussing a team member's skills, feedback, or action items:
- Call `person_note` to build their profile
- Call `delegate` if a task is assigned to them

## Behavior

- Do NOT ask permission to store — just do it silently alongside your main response
- Do NOT mention that you're storing unless the user asks
- Keep stored content concise — key facts, not full conversations
- Use relevant tags for easier retrieval later
